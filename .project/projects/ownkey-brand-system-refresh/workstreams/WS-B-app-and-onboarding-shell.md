# Workstream: WS-B App and Onboarding Shell

## Objective
Make setup, settings, and configuration screens feel like Ownkey while staying simple and readable.

## Owned Files/Areas
- Setup screens
- Settings screens
- About screen
- Voxtral/transcription settings
- App-level toast fallback styling

## Dependencies
- WS-A token and asset choices

## Risks
- Over-branding configuration screens can reduce scanability.

## Handoff Criteria
- Setup and settings use the Ownkey mark and palette consistently.
- Existing settings flows remain discoverable and readable at larger font scales.
