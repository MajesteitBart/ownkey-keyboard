# Google Play Store USP Draft

Working list for Play Store positioning. These are product claims based on the current Ownkey implementation and should be reviewed before publishing.

## Primary USP Set

1. Open-source Android keyboard with practical AI built in.
2. Bring your own API key: choose your provider instead of being locked into one account.
3. Voice dictation powered by configurable Mistral Voxtral ASR defaults.
4. Rewrite selected text from the keyboard with configurable LLM providers.
5. Provider presets for OpenAI, Anthropic, Mistral, OpenRouter, and custom endpoints.
6. API keys are stored encrypted on-device with Android Keystore-backed storage.
7. No added in-app monitoring of private keyboard content.
8. Improved EN/NL suggestions and autocorrect tuned for daily typing.
9. Clean dark Ownkey design with a focused setup and AI settings experience.
10. Based on FlorisBoard and released under Apache-2.0, with clear open-source attribution.

## Short Play Store Bullets

- AI typing without vendor lock-in
- BYOK provider control for dictation and rewrite
- Voxtral voice input built into the keyboard flow
- Rewrite text with OpenAI, Anthropic, Mistral, OpenRouter, or a custom endpoint
- Encrypted local API key storage
- Open-source, privacy-first Android keyboard
- Better suggestions and autocorrect for everyday typing
- Clean dark interface with Ownkey branding

## Strongest Positioning Angles

### Control
Ownkey gives you AI input on your terms: bring your own API key, choose your endpoint, and keep provider settings visible.

### Privacy
Ownkey does not add private-content monitoring inside the keyboard. Keys are stored encrypted on-device, and cloud AI requests only go to the endpoint you configure.

### Practical AI
Dictate, rewrite, and keep typing from the same keyboard instead of jumping between apps.

### Open Source
Ownkey is open-source and FlorisBoard-derived, so the keyboard can be inspected instead of treated as a black box.

## Copy Caveats

- Do not claim that cloud AI is local or fully private. Dictation and rewrite send request data to the configured provider when used.
- Do not imply that ChatGPT or Claude subscriptions replace API access.
- Do not mention a specific OpenAI model in store copy unless it has been rechecked near release.
- Avoid saying "secure" as a broad promise; use concrete implementation language like "encrypted on-device key storage."
